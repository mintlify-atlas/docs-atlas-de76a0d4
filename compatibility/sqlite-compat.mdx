## Overview

Turso aims to be fully compatible with SQLite, with opt-in features not supported by SQLite.

| Feature | Status | Notes |
|---------|--------|-------|
| SQLite file format | ✅ Full | Reads and writes the standard SQLite on-disk format |
| SQLite query language | 🚧 Partial | Most common statements and functions supported |
| SQLite C API | 🚧 Partial | Core prepare/step/column API supported; extensions not yet |

<Note>
Legacy SQLite databases are automatically converted to WAL mode when opened by Turso.
</Note>

## SQLite Journaling Modes

<Note>
Turso has no plans to support the rollback journal mode, as it locks the database file during writes. All rollback-type modes are marked as not needed.
</Note>

| Journal Mode | Status | Comment |
|--------------|--------|----------|
| wal | ✅ Yes | Default mode for new databases |
| mvcc | ✅ Yes | Turso-specific; enables concurrent transactions (experimental) |
| wal2 | ❌ No | Experimental feature in SQLite |
| delete | Not Needed | Locks database during writes |
| truncate | Not Needed | Locks database during writes |
| persist | Not Needed | Locks database during writes |
| memory | Not Needed | Locks database during writes |
